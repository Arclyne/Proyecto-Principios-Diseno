## CU-03 Inactivar Organización

### Descripción de caso de uso

| Atributo | Descripción |
| ------------------------ | ------------------------------------------------------------ |
| **ID**                   | CU-C-03                                                      |
| **Nombre**               | Inactivar Organización                                       |
| **Responsable**                    | Angel Gabriel Aguilar Hernández                              |
| **Descripción**          | Se cambia el estado de una o varias Organizaciones Vinculadas activas a inactivo en la base de datos. |
| **Disparador**           | El COORDINADOR selecciona la opción "Gestionar Organizaciones" en la **GUI_MenuCoordinador**. |
| **Actor primario**       | COORDINADOR                                                  |
| **Precondiciones**       | **Pre-1:** Existe al menos una **Organization** activa registrada en la base de datos. |
| **Postcondiciones**      | **Post-1:** Las **Organization** seleccionadas muestran estado "Inactivo" en el SPP. |
| **Flujo normal**         | **1.** El SPP recupera la lista de Organization activas (EX-01), cierra la GUI_MenuCoordinador y muestra la GUI_ManageOrganizations con la lista, las casillas de selección y la opción 'Inactivar. <br />**2.** El COORDINADOR selecciona una o varias **Organization** marcando sus casillas.(Ver FA-01)<br />**3.** El COORDINADOR selecciona la opción "Inactivar".<br />**4.** El SPP recopila los identificadores de las **Organization** seleccionadas. <br />**5.** El SPP cambia el estado de las **Organization** seleccionadas a "Inactivo" en la base de datos. (EX-02) <br />**6.** El SPP muestra la **GUI_MessageSuccess** con el mensaje "Las organizaciones se inactivaron correctamente" y la opción "Aceptar". <br />**7.** El COORDINADOR selecciona la opción "Aceptar". <br />**8.** El SPP cierra la **GUI_MessageSuccess** y recarga la **GUI_ManageOrganizations** mostrando solo las **Organization** activas restantes. <br />9. Fin del Caso de uso. |
| **Flujos alternos**      | **FA-01: Sin selección** <br />**1.** El COORDINADOR selecciona la opción "Inactivar" sin haber seleccionado ninguna **Organization**. <br />**2.** El SPP muestra la **GUI_MessageWarning** con el mensaje "Debe seleccionar al menos una organización" y la opción "Aceptar". <br />**3.** El COORDINADOR selecciona la opción "Aceptar". <br />**4.** El SPP cierra la **GUI_MessageWarning**. **5.** El flujo continúa en el paso 2 del FN. |
| **Excepciones**          | **EX-01: Error al recuperar las organizaciones activas**<br/> **1.** El SPP no logra recuperar la lista de **Organization** activas (la consulta falla o hay un problema de conexión con la base de datos).<br/> **2.** El SPP muestra la **GUI_MessageError** con el mensaje "No se pudieron recuperar las organizaciones. Por favor intente de nuevo más tarde." y la opción "Aceptar".<br/> **3.** El COORDINADOR selecciona la opción "Aceptar".<br/> **4.** El SPP cierra la **GUI_MessageError**.<br/> **5.** Fin del caso de uso.<br />**EX-02: Error al inactivar** <br />**1.** Ocurre un error al ejecutar la inactivación o un problema de conexión con la base de datos. <br />**2.** El SPP muestra la **GUI_MessageError** con el mensaje "No se pudieron inactivar las organizaciones. Por favor intente nuevamente en unos minutos." y la opción "Aceptar". <br />**3.** El COORDINADOR selecciona la opción "Aceptar". <br />**4.** El SPP cierra la **GUI_MessageError**. <br />**5.** El flujo continúa en el paso 2 del FN. |
| **Reglas de validación** | **RV-01:** Debe seleccionarse al menos una **Organization** para ejecutar la inactivación. **RV-02:** Solo pueden inactivarse **Organization** que se encuentren en estado "Activo". <br />**RV-03:** El cambio de estado debe persistirse en la base de datos para considerarse exitoso. |
